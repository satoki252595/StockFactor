# JP Doubling Screener — 日本株「半年で2倍」候補の日次抽出

## 目的
直近5年で **126営業日(≒半年)以内に+100%** となった日本株（doublers）を分析し、
点火前に共通して現れる **本質的要素** をデータで検証して採用。現在の全上場銘柄を同じ要素で
機械採点し、**何個の本質的要素が当てはまるか** で順位付けする。Python が定量計算を担い、
Claude（このスキル）が定性統合して最終ウォッチリストを作る。**ランニングコスト0円。**

## 前提・制約
- データ: yfinance（無料）。ユニバース: JPX 公開リスト（`data/universe.csv`）。
- ネットワーク: routine 環境を **Custom**（`*.finance.yahoo.com` 等を許可）か **Full** にする
  （`docs/operations.md` 参照）。既定 Trusted では金融ホストが遮断される。
- ファンダ要素は **現時点スナップショット**（`.info`）のみ。過去点別データは無料で入手不可なため
  過去検証は限定的（テクニカル・マクロは価格履歴で検証済み）。
- 投資助言ではなく要素該当度のスクリーニング支援。

## 本質的要素カタログ（採点対象 — 実データ検証済み）
> **重要**: 当初の「米国流モメンタム/ブレイク型」仮説は実データで**棄却**された（`FINDINGS.md`）。
> 日本の半年2倍株は **反転（リバーサル）型**＝「高ボラ × 高値から下落（出遅れ/押し目）×
> 出来高急増 × ボラ拡大 × 直近は投げ」。下表は要約で、`src/stockfactor/score.py` の `RULES`
> を唯一の正とする。**ルールを変えたら必ず experiment ワークフローで再検証すること。**

### テクニカル（実データで識別力を確認, validated=True）
- **high_volatility**: ATR% ≥ 0.04（最強の識別子, AUC 0.81 / lift +0.45）
- **off_highs**: 終値 ≤ 52週高値×0.80（高値から下落・出遅れ位置, lift +0.27）
- **volume_surge**: 直近5日出来高 ≥ 60日平均×1.3（点火の出来高, lift +0.22）
- **below_sma25**: 終値 ≤ 25日線（押し目/底値圏, lift +0.22）
- **vol_expansion**: 直近ボラ拡大 vcp ≥ 1.0（収縮ではない, lift +0.19）
- **recent_dip**: 直近1カ月リターン ≤ 0（反転直前の投げ/capitulation, AUC 0.37）

### ファンダ×ミクロ（過去検証不可→フォワード適用, validated=False）
> 無料では決算の発表時点データが無く過去検証できない。文献的根拠でフィルタとして加点。
- 小型時価総額（500億円未満）= 伸びしろ／増収率15%以上／増益20%以上／ROE10%以上／PSR5倍以下

### マクロ（本サンプルでは識別力弱→低ウェイトの参考）
- 小型株（グロース市場）が市場を6Mで上回る（lift ほぼ0）。市場レジームは AUC≈0.5 で非採点。
- → マクロは採点よりも、定性評価で「順張り/逆張りの地合い」の所見として用いる。

## 日次手順（このスキルが実行する）

1. **データ更新の確認**: `data/universe.csv` が無ければ
   `python -c "from src.stockfactor import universe as u; u.refresh()"`（月1で十分）。
2. **決定論採点を実行**:
   ```bash
   python run_daily.py            # 全銘柄（routine 本番）
   # 動作確認時は: python run_daily.py --limit 300
   ```
   → `output/daily_scores.csv`（全候補・要素内訳）と `output/daily_report.md`（上位30）を生成。
3. **定性統合（LLMパート＝あなたの仕事）**: `output/daily_scores.csv` の上位を読み、各銘柄について:
   - 充足要素の**内訳（テクニカル/ファンダ/マクロのバランス）**を見る。3群すべてに該当する銘柄を重視。
   - 不自然な値（急騰直後で既に2倍済み＝出遅れでない、出来高が異常な仕手的動き等）を除外。
   - セクター・テーマの妥当性、`hits` に挙がった要素が**互いに整合**しているかを確認。
   - 同種の過去 doubler（`experiments/results/doublers.csv`）との類似性があれば言及。
4. **ウォッチリスト出力**: `output/watchlist_<YYYY-MM-DD>.md` に上位10〜20銘柄を、
   - コード・銘柄名・終値・充足要素数（群別内訳）・該当要素・**一言の根拠**・**留意点**
   で表にまとめる。最後に「市場レジーム（マクロ）の所見」を1段落。
5. （任意）結果をコミット。

## 日次レポートを Notion に生成する（LLMパートの本命・Claudeルーチン向け）
GitHub Actions は生スコア（`output/daily_scores.csv`＋Notion「日次スクリーニング結果」DB）を毎日生成する。
**LLMリサーチ付きレポートは Claude が担当**し、Notion「StockFactor 日次レポート」DB に1日1行で投入する。

- 投入先: Notion データベース **「StockFactor 日次レポート」**
  （data_source_id: `6d055ec2-093f-4cdf-b306-51ca97c5ac8c` / 親ページ「小型成長株の推し銘柄」）。
  Notion コネクタ経由で書き込む（ルーチンならトークン不要）。
- 手順:
  1. `python run_daily.py` を実行（または最新の `output/daily_scores.csv` を読む）。
  2. **上位10銘柄**について、各社の**事業内容・ファンダ（売上/利益成長・ROE・時価総額・割安度）・強み・留意点**を
     `WebSearch`／各社IR で簡潔にリサーチ。
  3. その日の `日付` で1ページ作成（properties: タイトル=「YYYY-MM-DD 半年2倍候補レポート（上位10）」,
     日付, 最高要素数, 候補数, 市場メモ）。本文 = サマリー表＋順位別の解説（スコア内訳T/F/M・該当要素・
     事業・ファンダ・強み・留意点）＋末尾に評価ロジック/出所/ディスクレーマー。
  4. 同じ日付が既にあれば作り直さない（重複防止）。
- フォーマットは 2026-06-09 のレポートを雛形にする。

## 重要な運用ルール
- **しきい値の根拠はデータ**: 体感で閾値を動かさない。変更は score.py → experiment 再検証 → 反映。
- **2群以上に該当しない銘柄は原則採用しない**（テクニカルだけの急騰は誤検出が多い）。
- 既に半年で2倍**済み**の銘柄は「これから2倍」候補ではない。初動段階（過熱前）を重視。
