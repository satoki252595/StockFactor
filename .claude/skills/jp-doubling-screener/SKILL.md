# JP Doubling Screener — 日本株「半年で2倍」候補の日次抽出

## 目的
全上場銘柄・約6年の履歴で **126営業日(≒半年)以内に+100%** となった日本株（doublers）を分析し、
点火前に現れる **本質的要素** をデータで検証して採用。現在の全銘柄を同じ要素で機械採点する。
2倍株は **モメンタム型（高値ブレイク・順張り, 43%）** と **反転型（押し目・逆張り, 57%）** の
2モードに分かれるため、共通要素＋型別トラックで採点し `setup_type` を判定する。
Python が定量計算を担い、Claude（このスキル）が型を踏まえ定性統合して最終ウォッチリストを作る。
**ランニングコスト0円。**

## 前提・制約
- データ: yfinance（無料）。ユニバース: JPX 公開リスト（`data/universe.csv`）。
- ネットワーク: routine 環境を **Custom**（`*.finance.yahoo.com` 等を許可）か **Full** にする
  （`docs/operations.md` 参照）。既定 Trusted では金融ホストが遮断される。
- ファンダ要素は通常 **現時点スナップショット**（`.info`）。過去検証は yfinance の四半期決算
  （`quarterly_income_stmt`）で **point-in-time 近似**が可能（`compute_fundamental_pit` /
  実験は `--with-pit-funda`、開示遅延+50日で近似）。より精密には J-Quants（`jquants.py`、要認証）。
- 投資助言ではなく要素該当度のスクリーニング支援。

## 本質的要素カタログ（採点対象 — 全銘柄3,734・正例1,233件で検証済み）
> **重要（v2で更新）**: 日本の半年2倍株には **2つのモード** がほぼ拮抗して存在する（`FINDINGS.md`）。
> 旧 v1（500サンプル）の「反転型のみ・モメンタム型は棄却」は **標本バイアスによる誤り**で、
> 全銘柄検証で **モメンタム型が43%（キオクシア型）** を占めることが判明した。
> `src/stockfactor/score.py` を **2トラック採点** に再設計済み。下表は要約で、score.py の
> ルールを唯一の正とする。**ルールを変えたら必ず experiment ワークフローで再検証すること。**

### 共通要素（両モード必須・validated=True）
- **high_volatility**: ATR% ≥ 0.04（最強の識別子・両型共通, AUC 0.78 / lift +0.39）
- **volume_surge**: 直近5日出来高 ≥ 60日平均×1.3（点火の出来高, lift +0.17）
- **vol_expansion**: ボラ拡大 vcp ≥ 1.0（収縮ではない, lift +0.14）

### モメンタム型トラック（高値ブレイク・順張り — doublerの43%, validated=True）
- **uptrend_200**: 終値 > 200日線（上昇トレンド。型の最強分離器, モメンタム99.6% / 反転10%）
- **trend_aligned**: 25日>75日>200日（パーフェクトオーダー, モメンタム87% / 反転10%）
- **near_highs**: 終値 ≥ 52週高値×0.85（高値圏, モメンタム0.88 / 反転0.56）
- **positive_rs**: 対TOPIX 6M相対強さ ≥ +10%（モメンタム+0.52 / 反転-0.19）
- **momentum_up**: 直近1カ月リターン ≥ 0（モメンタム+0.18 / 反転-0.15）

### 反転型トラック（押し目・逆張り — doublerの57%, validated=True）
- **off_highs**: 終値 ≤ 52週高値×0.80（高値から下落・出遅れ位置, lift +0.24）
- **below_sma25**: 終値 ≤ 25日線（押し目/底値圏, lift +0.22）
- **recent_dip**: 直近1カ月リターン ≤ 0（反転直前の投げ/capitulation, lift +0.19）
- **deep_drawdown**: 52週高値から25%以上下落（十分な押し目, pos中央値-0.28）

> 採点は各銘柄で「共通+モメンタム型」「共通+反転型」を別々に計算し、**高い方を primary**、
> `setup_type` で型を明示する。off_highs と near_highs は相反するため単純合算しない。

### ファンダ×ミクロ（両型共通・validated=False / PIT検証は実装済み）
> yfinance `.info` は現時点スナップショット。過去 point-in-time 検証は
> `quarterly_income_stmt`（`--with-pit-funda`）で近似可能（開示遅延+50日）。
- 小型時価総額（500億円未満）= 伸びしろ／増収率15%以上／増益20%以上／ROE10%以上／PSR5倍以下

### マクロ（識別力弱→低ウェイトの参考）
- 小型株（グロース市場）が市場を6Mで上回る（lift ほぼ0）。市場レジームは AUC≈0.5 で非採点。
- → マクロは採点よりも、定性評価で「順張り/逆張りの地合い」の所見として用いる。

## 日次手順（このスキルが実行する）

1. **データ更新の確認**: `data/universe.csv` が無ければ
   `python -c "from src.stockfactor import universe as u; u.refresh()"`（月1で十分）。
2. **決定論採点を実行**:
   ```bash
   python run_daily.py            # 全銘柄（routine 本番）
   # 動作確認時は: python run_daily.py --limit 300
   ```
   → `output/daily_scores.csv`（全候補・`setup_type`・要素内訳）と
     `output/daily_report.md`（モメンタム型／反転型の型別上位）を生成。
3. **定性統合（LLMパート＝あなたの仕事）**: `output/daily_scores.csv` の上位を読み、各銘柄について:
   - **`setup_type`（momentum / reversal）で分けて評価する**。レポートは型別に上位が出る。
   - 充足要素の**群別内訳（テクニカル/ファンダ/マクロ）**を見る。3群すべてに該当する銘柄を重視。
   - **型と要素の整合**を確認: モメンタム型なら uptrend_200・near_highs・positive_rs が揃うべき、
     反転型なら off_highs・below_sma25・recent_dip が揃うべき。混在は弱いシグナル。
   - 不自然な値を除外: 反転型で既に2倍済み（出遅れでない）／モメンタム型で過熱しすぎ
     （短期に急騰し既に+80%等）／出来高が異常な仕手的動き。
   - 同種の過去 doubler（`experiments/results/doublers.csv`）との類似性があれば言及。
4. **ウォッチリスト出力**: `output/watchlist_<YYYY-MM-DD>.md` に**両型から**上位を、
   - コード・銘柄名・終値・**型(setup_type)**・充足要素数（群別内訳）・該当要素・**一言の根拠**・**留意点**
   で表にまとめる。モメンタム型・反転型でセクションを分けるとよい。
   最後に「市場レジーム（マクロ）の所見」を1段落（順張り地合いならモメンタム型を厚めに）。
5. （任意）結果をコミット。

## 重要な運用ルール
- **しきい値の根拠はデータ**: 体感で閾値を動かさない。変更は score.py → experiment 再検証 → 反映。
- **共通要素(high_volatility等)＋型トラックの両方に該当する銘柄を重視**（型が曖昧な銘柄は誤検出が多い）。
- 既に半年で2倍**済み**の銘柄は「これから2倍」候補ではない。初動段階（過熱前）を重視。
- **★ モメンタム型と反転型で選別基準を変える（非対称性, FINDINGS.md）**:
  - **モメンタム型**は分類自体は doubler の証拠にならない（負例にも多い）。**スコア(要素充足数)が高い
    銘柄だけを厳選**する。uptrend_200 だけでなく near_highs + positive_rs + volume_surge +
    ファンダまで揃ったものを採る。過熱（短期急騰で既に大幅高）は除外。
  - **反転型**はシグナル自体が選択的（正例に偏る）。スコア中位でも拾う価値がある。
    押し目・出来高急増・直近投げが揃っているかを重視。
