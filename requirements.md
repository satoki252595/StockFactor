# StockFactor 要件一覧（優先度順）

直近5年で **半年（≒126営業日）以内に株価が2倍以上** になった日本株を抽出し、その「本質的要素」を
テクニカル / ファンダメンタルズ×ミクロ / マクロ の観点で定量化。日次バッチで現在の全上場銘柄を
同じ要素で機械採点し、Claude スキルが定性統合して候補リストを出す仕組み。

**絶対制約: ランニングコスト 0 円。**

---

## 0. 決定事項（ユーザー合意済み）

| 項目 | 決定 |
| --- | --- |
| 日次バッチ実行基盤 | **Claude Code Web スケジュール（routine）** |
| LLM 定量評価の0円実現 | **決定論 Python（機械計算）＋ Claude スキル（定性統合）** |
| 対象ユニバース | **流動性フィルタ後の全上場** |

### 0円運用のアーキテクチャ含意
- データ取得は **yfinance（Yahoo Finance, 無料）**。ユニバースは **JPX 公開リスト（無料）**。
- 重い計算（取得・指標算出・採点）は**決定論 Python**で実施＝常時0円。
- 「LLM評価」= Claude スキルが採点結果を読み、要素定義に沿って定性統合（既存サブスク内＝追加0円）。
- **開発中の仮説検証は GitHub Actions（無料枠／パブリックリポは無制限）を実験ハーネスとして使用。**
- ⚠️ Claude Code Web スケジュール実行で yfinance を使うには、環境のネットワークを
  **Custom**（`query1.finance.yahoo.com`, `query2.finance.yahoo.com`, `*.finance.yahoo.com`, `fc.yahoo.com`
  を追加＋既定パッケージ許可リスト併用）か **Full** に設定する必要がある（既定 Trusted では金融ホストが遮断される）。

## 1. 要件（優先度: P0=必須/基盤, P1=重要, P2=拡張）

### P0 — 基盤（これが無いと何も始まらない）
- **R1. データ取得基盤**: yfinance で日本株 OHLCV（5年・日足）を無料取得。リトライ・キャッシュ・レート制御。
- **R2. ユニバース管理**: JPX 上場銘柄一覧を取得しリポジトリにコミット（ネット不要で再利用）。流動性フィルタ
  （平均売買代金・株価・上場期間）を適用。小型株は2倍株の主戦場なので残しつつノイズ除去。
- **R3. 2倍株スクリーナ**: 各銘柄の価格系列から「ある起点 t0 から126営業日以内に+100%以上」となる
  *点火点 t0* を検出（先読みなし・ラベルのみ未来参照）。直近5年で全イベント抽出。
- **R4. 仮説検証ハーネス**: GitHub Actions で実データ実験を回し、結果（doublers, 要素統計, レポート）を
  リポジトリへコミット。合成データでロジックの単体テスト（ネット不要・コンテナ内実行）。

### P1 — 要素の定量化と検証（本質的要素の抽出）
- **R5. テクニカル要素**: 52週高値接近/ブレイク, 出来高急増, 移動平均整列/GC, モメンタム(3/6/12M),
  ボラ収縮(VCP的), TOPIX相対強さ。**→ 価格履歴のみで過去検証可能。**
- **R6. マクロ要素**: 市場レジーム（TOPIX 200日線上/下）, 市場6Mリターン, 小型株レジーム,
  セクター・モメンタム。**→ 価格履歴のみで過去検証可能。**
- **R7. ファンダ×ミクロ要素**: 小型時価総額, 増収率, 増益/黒字転換, ROE/マージン改善, 割安成長(PSR/PEG),
  成長セクター。**→ 無料の過去ファンダ点別データが無いため過去検証は限定的。フォワード採点＋文献根拠＋
  現時点スナップショット検証で扱う（この制約は明記する）。**
- **R8. 要素の有効性検証（backtest）**: doubler 群 vs 対照群（非イベント点）で各要素の分布差・
  情報量（lift, 命中率, t検定/AUC）を測定し、**識別力のある要素だけを採用**。閾値もデータから決定。
- **R9. スコアリング**: 採用要素の充足数（重み付き）を算出＝「本質的要素が何個当てはまるか」。

### P2 — 運用・自動化・スキル化
- **R10. 日次バッチ本体** (`run_daily.py`): ユニバース→取得→流動性フィルタ→全要素採点→
  `output/daily_scores.csv` と `output/daily_report.md` 出力。
- **R11. Claude スキル**: 要素定義＋閾値＋採点結果の読み方＋定性統合手順を `SKILL.md` に記述。
  日次でスキルを起動→Python採点→Claude定性評価→ウォッチリスト確定→リポジトリにコミット。
- **R12. スケジュール運用ドキュメント**: Claude Code Web routine 設定とネットワーク設定手順。
- **R13. 結果の蓄積/モニタリング**: 日次出力を履歴保存し、採点上位銘柄のその後を追跡して
  要素の有効性を継続検証（フィードバックループ）。

## 2. 実装順序（重要なものから）

1. R1, R2, R3, R4（基盤＋検証ハーネス）← まず実データで「取得できる/2倍株を検出できる」を証明
2. R5, R6, R8（テクニカル・マクロ要素を *データで* 検証し、有効要素を確定）
3. R7, R9（ファンダ要素を追加し、総合スコア定義）
4. R10, R11, R12（日次バッチ＋スキル＋運用手順）
5. R13（継続検証）

---

## 3. 仮説（検証対象）

- **H1**: yfinance で日本株5年日足を無料・安定取得できる。【R4で検証】
- **H2**: 直近5年で「126営業日内2倍」イベントは十分な数（統計に足る）存在する。【exp02】
- **H3**: 点火点 t0 直前に、テクニカル/マクロ要素が対照群より有意に偏る（識別力がある）。【exp03/R8】
- **H4**: 2倍株は小型株に偏る（時価総額が対照より有意に小さい）。【exp03】

各仮説は `experiments/` の小規模実験で検証してから採用・閾値決定する。

---

## 4. 非目標 / 既知の限界

- 無料では **過去の点別ファンダ（決算の発表時点データ）** が入手困難 → ファンダ要素の厳密な過去検証は不可。
  文献的根拠＋フォワード適用＋現時点スナップショットで補完し、限界を明示する。
- 銘柄の上場廃止・統合等による **サバイバーシップ・バイアス** はユニバースの取り方で部分的に残る（明記）。
- 本システムは投資助言ではなく、要素該当度のスクリーニング支援である。
