# 検証結果: 日本株「半年で2倍」の本質的要素

> 実データ（yfinance, 直近約6年）で仮説を検証した結論。再現は GitHub Actions の `experiment`
> ワークフロー（無料）。生データは `experiments/results/`。
> **v2 (全銘柄3,734・正例1,233件) で結論を更新**: 2倍株には「モメンタム型」と「反転型」の
> 2つのモードが拮抗して存在する。旧 v1（500サンプル）の「反転型のみ」は標本バイアスだった。

## 実験設定 (v2)
- ユニバース: JPX 全 **3,734銘柄**（v1 は 500サンプル）。並列処理で全数検証。
- 2倍株定義: 起点 t0 から **126営業日以内に終値+100%**（先読みはラベルのみ、要素は t0 まで）。
- 正例: doubler の点火点 t0（**1,233件**）／負例: 将来126日で+50%未満（6,216件）。
- 指標: **AUC**（0.5=無情報）と **lift**（正例命中率−負例命中率）、**K-Means クラスター分析**。

## H2 ✅ 2倍株は豊富に存在する
- doubler イベント **1,233件**、2倍を経験した銘柄 **808 / 3,734（21.6%）**。
- 全銘柄でも約2割が半年2倍を経験 → サンプルでなく母集団レベルで普遍的な現象。

## ★ 最重要発見: 2倍株には2つのモードがある（v2で判明）
K-Means(k=2) で正例1,233件を分類すると、**ほぼ拮抗する2クラスター**に明確に分かれた
（`experiments/results/cluster_analysis.csv`）。

| 特徴量 | モメンタム型(526件/43%) | 反転型(707件/57%) | 差 |
| --- | ---: | ---: | ---: |
| dist_52w_high（高値接近度） | **0.88**（高値圏） | 0.56（高値から44%下） | +0.32 |
| above_sma200（上昇トレンド） | **99.6%** | 10.3% | +0.89 |
| sma_aligned（パーフェクトオーダー） | **87.5%** | 10.0% | +0.77 |
| rs_6m（対TOPIX相対強さ） | **+0.52** | -0.19 | +0.71 |
| ret_1m（直近1カ月） | **+0.18** | -0.15 | +0.33 |
| vol_ratio（出来高急増） | 1.86 | 1.12 | +0.74 |
| atr_pct（ボラティリティ） | 0.053 | 0.051 | **+0.002（共通）** |

- **モメンタム型**（例: キオクシア型）= 52週高値圏 × 上昇トレンド × パーフェクトオーダー ×
  市場を大幅アウトパフォーム中に、さらに2倍へ。**高値ブレイク・順張り**。
- **反転型** = 高値から下落 × 押し目 × 直近投げ × 市場アンダーパフォームから反転。**出遅れ・逆張り**。
- **両型の唯一の共通必須条件は `high_volatility`（atr_pct≥0.04）**。値はほぼ同一（0.053 vs 0.051）。
  → 「動ける銘柄であること」だけが2倍株の絶対条件で、その上で2つの異なる経路がある。

## なぜ v1 では反転型しか見えなかったか
- v1 は 500銘柄サンプル（seed=42）。標本に反転型 doubler が多く入り、平均すると反転型の
  特徴（高値から下落・押し目）が支配的に見えた。
- 全銘柄ではモメンタム型が43%を占め、**無視できない**。旧スコア（反転型ルールのみ）は
  モメンタム型2倍株を構造的に取りこぼしていた。

## スコア設計 (R11): 2トラック採点
`src/stockfactor/score.py` を2トラックに再設計:
- **共通要素**（high_volatility, volume_surge, vol_expansion）: 両トラックで加点
- **反転型トラック**: off_highs, below_sma25, recent_dip, deep_drawdown
- **モメンタム型トラック**: uptrend_200, trend_aligned, near_highs, positive_rs, momentum_up
- **ファンダ/マクロ**: 両型共通で加点

各銘柄で「共通+反転型」「共通+モメンタム型」を別々に採点し、**高い方を primary**、
`setup_type` で型を明示。off_highs（反転）と near_highs（モメンタム）は相反するため、
単純合算ではなく **トラック別 max** を取る設計が必須。

### 検証: ルールベース分類は K-Means と一致
score.py のルール判定（momentum 40.6%）が独立手法の K-Means（43%）とほぼ一致。
分類後の各型の特徴量（dist_52w_high, above_sma200, rs_6m）も完全に分離 → 設計の妥当性を確認。

## ファンダ×ミクロの扱い
- yfinance `.info` は現時点スナップショットのみ。過去 point-in-time 検証は不可。
- v2 で **yfinance `quarterly_income_stmt`** を使った point-in-time 近似を実装
  （`factors.build_quarterly_pit` / `compute_fundamental_pit`、開示遅延+50日で近似）。
  `--with-pit-funda` フラグで実験に組み込み、過去の増収増益・ROE を t0 時点で評価可能。
- 別途 J-Quants API クライアント骨格（`jquants.py`）も用意。認証情報があれば
  より正確な開示日(DisclosedDate)ベースの検証が可能。

## 重要な注意（誤用防止）
- 本要素は「2倍に**なった**銘柄の点火前の姿」を非上昇銘柄と対比して抽出したもの。
  同じ姿でも**下落する銘柄も多い**（高ボラはリスクも高い）。勝率・下落リスクは別問題。
- 2モードの発見は単一期間(6年)・126日地平での結果。期間分割・地平変更での頑健性は今後の課題。
